# Escopo do MVP

O MVP do TRACEFLOW tem como objetivo demonstrar a cadeia principal de rastreabilidade:

```txt
Requisito -> Tarefa -> Artefato GitHub
```

## Entra no MVP

- Cadastro de projeto.
- Configuracao de repositorio GitHub.
- Sincronizacao e importacao de commits, pull requests e issues.
- Cadastro de requisitos.
- Cadastro de tarefas.
- Kanban simples.
- Vinculos manuais de rastreabilidade.
- Consulta da cadeia Requisito -> Tarefa -> Artefato GitHub.

## Fora do MVP inicial

- Login completo.
- Recuperacao de senha.
- Gestao completa de usuarios.
- Perfis de acesso avancados.
- Casos de teste.
- Defeitos.
- Alertas.
- Notificacoes.
- Relatorios PDF.
- Dashboards avancados.
- Associacao automatica de commits.

## Justificativa

O objetivo do MVP e validar a proposta central do TRACEFLOW sem aumentar demais a complexidade tecnica. As funcionalidades fora do MVP ficam previstas para evolucao futura.
